HttpLoadTester

Учебное приложение для нагрузочного тестирования HTTP-сервисов с графическим интерфейсом (WPF).

Что реализовано
GUI-приложение на WPF для удобного запуска тестов
Отправка HTTP-запросов (GET/POST и др.)
Настройка количества запросов и уровня нагрузки
Параллельное выполнение запросов (нагрузочное тестирование)
Логирование результатов выполнения
Отображение статусов ответов сервера
Базовая обработка ошибок соединения

Архитектурная идея

Приложение построено как монолитный desktop-инструмент:

UI (WPF) — отвечает за ввод параметров тестирования и отображение результатов
Логика нагрузки — выполняет многопоточные HTTP-запросы
Сетевой слой — использует HTTP-клиент для отправки запросов
Логирование — фиксирует результаты выполнения тестов

Основная цель — быстрое тестирование API и сервисов под нагрузкой без сложной настройки.

Пример сценария использования
Ввести URL (например: http://localhost:5000/api/test)
Выбрать метод (GET/POST)
Указать:
количество запросов (например, 1000)
уровень параллелизма (например, 50 потоков)
Запустить тест
Анализировать результаты и логи
Ограничения текущего MVP
Нет детальной аналитики (графиков, percentiles, latency distribution)
Нет поддержки сценариев (только одиночные запросы)
Ограниченное логирование
Нет экспорта результатов
Нет поддержки авторизации (JWT, OAuth и т.д.)
Отсутствует конфигурация через файлы (JSON/YAML)
